# Writing Guidelines

## Core Rules

- Clarity over cleverness.
- Invitation over instruction.
- Observation over judgment.
- Specific privacy language over reassuring vagueness.
- One idea per sentence when the user is making a decision.
- Fewer words on screen than feel possible.

## Product Copy

Use product copy to help the user act. Do not use it to explain the brand.

Good product copy:

- Names the object: manual, answer, artifact.
- Names the state: included, private, omitted.
- Names the consequence: visible in shared link, kept in this browser.
- Names the next action: copy, export, continue, review.

Weak product copy:

- Repeats the heading.
- Praises the user for routine actions.
- Uses abstract reassurance.
- Adds a second metaphor.

## Labels

Labels should be short and stable. Avoid changing the same concept across screens.

Use:

- Memory only
- Save in link
- Included
- Private
- Omitted
- Share link
- Export

Avoid:

- Local mode
- Encoded payload
- Hidden
- Public
- Suppressed
- Publish

Note: "Public" can be used in technical code, but user-facing copy should prefer "included" or "visible in the shared manual".

## Help Text

Help text should answer the question the user is likely asking at that moment.

Pattern:

1. What this setting does.
2. What happens to the user's words.
3. What action remains available.

Example:

"Save in link stores answers inside the URL. Anyone with that link can open the included manual."

## Errors

Use this structure:

1. What did not work.
2. What happened to the user's answers.
3. What to try next.

Example:

"The share link could not be copied. Your manual has not changed. Select the link text and copy it manually."

## AI Copy

When AI support appears, describe it as optional assistance. Never imply certainty, diagnosis, or authority.

Use:

- Suggested draft
- Try this wording
- You can edit anything

Avoid:

- AI detected
- Your true style is
- Best answer
- Final profile

## Microcopy Checklist

Before shipping copy, check:

- Does this use canonical terms?
- Does it make the privacy consequence clear?
- Does it avoid pressure?
- Does it avoid clinical or gamified language?
- Can one sentence be removed?
