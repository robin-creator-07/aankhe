# Glossary

## Canonical Product Terms

### Ankahe

The product name. Ankahe helps people create private personal or professional manuals that can be shared deliberately.

### Manual

The structured document the user creates. Use "manual" for the main object, not profile, report, assessment, or dashboard.

### Me Manual

A personal manual focused on the user's general communication, needs, boundaries, and context.

### Work Manual

A professional manual focused on collaboration, feedback, working style, and team communication.

### Answer

The user's response to a prompt. Use "answer", not entry, data point, result, or input.

### Visibility

The user's choice about whether an answer appears in the shared artifact.

### Included

An answer marked to appear in the shared manual or export.

### Private

An answer kept out of the shared artifact. Private answers should be described as staying in this browser unless the storage mode says otherwise.

### Omitted

An answer intentionally left out. Omission is not a failure state.

### Artifact

The finished manual surface that can be exported, copied, shown, or shared.

### Memory only

Storage mode where answers remain in browser memory and disappear when the session is lost or refreshed.

### Save in link

Storage mode where answers are encoded into the URL. This is convenient, but anyone with the link can open what it contains.

### Share link

A link to the included manual. Copy should state what the link includes.

### QR code

A scannable form of the share link. It carries the same visibility rules as the link.

### Export

The act of creating a PDF, image, or other portable artifact from the included manual.

### Sample manual

A non-user example used to show the product. Samples must never imply that the product knows the user's identity or truth.

## Use With Care

### Public

Avoid in user-facing copy when it could imply discoverability on the open web. Prefer "included" or "visible in the shared manual".

### Profile

Avoid for the main object. It can feel like a fixed identity record. Prefer "manual".

### Hidden

Avoid as the opposite of included. Prefer "private" because it names the user's intention without sounding secretive.

## Banned Or Strongly Discouraged

Do not use these for Ankahe product surfaces:

- Diagnosis
- Assessment
- Score
- Optimize
- Unlock
- Magic
- XP
- Streak
- Level
- Treatment
- Therapy
- True self
- Personality verdict
